1. Basic Document Skeleton

Every HTML document follows a basic structure that includes:

  • <!DOCTYPE html> → Declares that the document uses HTML5
  • <html> ... </html> → The root element wrapping everything
  • <head> ... </head> → Holds metadata, title, links, and scripts
  • <body> ... </body> → Contains all the visible content

2. Explanation of Parts

<!DOCTYPE html>
Defines the type of document and version of HTML being used.
<html>
The root element that encloses the entire web page.
<head>
Includes page title, meta information, links to stylesheets, and scripts.
<title>
Sets the page title shown on the browser tab.
<meta>
Provides information such as character encoding and viewport settings.
<body>
Contains all content that users see, like headings, paragraphs, images, and links.

3. Semantic Structure

To improve readability and SEO, HTML provides semantic tags for layout:

  • <header> → Top section (logo, navigation)
  • <nav> → Defines navigation links
  • <main> → Main content area
  • <section> → Groups related content
  • <article> → Independent block (like a blog post)
  • <aside> → Side content such as notes or ads
  • <footer> → Bottom section (credits, copyright)

4. Recommended Directory Structure

Organizing your project files is crucial for maintainability. A common structure is:

/project-root
  /assets
    /css
      styles.css
    /js
      scripts.js
    /images
      logo.png   
    /pages   
      index.html
      tags.html 
      structure.html
      Forms and Table.html

This keeps your HTML files separate from styles, scripts, and images.

When the projects becomes too large, messy structure would bring diffuculties . so take time, plan , strucuture and document your work.
